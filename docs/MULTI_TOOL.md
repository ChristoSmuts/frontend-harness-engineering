# Multi-tool harness guide

This toolkit targets **any coding agent** that supports thin project docs (`AGENTS.md`), portable **skills** (`SKILL.md`), and optional rules/hooks. Cursor is fully supported out of the box; Claude Code, Codex CLI, and Gemini CLI use the same templates with different on-disk paths.

## Quick reference

| Tool | Always-on context | Skills | Rules | Stop verification |
|------|-------------------|--------|-------|-------------------|
| **Cursor** | `AGENTS.md` | `.cursor/skills/` | `.cursor/rules/*.mdc` | `.cursor/hooks.json` |
| **Claude Code** | `AGENTS.md` + optional `CLAUDE.md` | `.claude/skills/` | `.claude/rules/*.md` | Skill `frontend-verify` or manual commands |
| **Codex CLI** | `AGENTS.md` | `.agents/skills/` | `AGENTS.md` (keep thin) | Optional `.codex/` hooks when project is trusted |
| **Gemini CLI** | `AGENTS.md` + optional `GEMINI.md` | `.agents/skills/` or `.gemini/skills/` | `AGENTS.md` / `GEMINI.md` | Skill `frontend-verify` or manual |

Path details: [manifest/TOOL_LAYOUT.md](../manifest/TOOL_LAYOUT.md). Emit modes: [EMIT_STRATEGIES.md](EMIT_STRATEGIES.md).

**Canonical skills (multi-tool `full`):** `.agents/skills/` — mirrors under `.cursor/skills/` and `.claude/skills/`.

## Bootstrap with any agent

You do **not** need Cursor to run bootstrap.

1. Open the **target frontend repo** in your tool (IDE, terminal agent, etc.).
2. Paste [prompts/MASTER_BOOTSTRAP.md](../prompts/MASTER_BOOTSTRAP.md) into the session.
3. Tell the agent which tools you use, for example:

   ```text
   AI tools: Cursor + Codex CLI. Templates at <path-to-frontend-harness-engineering>/templates/
   Run intake, then generate harness for those tools only.
   ```

4. Approve the Phase B plan, then generate.

The agent should write skills to **canonical** `.agents/skills/` first, then mirror with `scripts/sync-skills.sh` on `full` emit.

## Per-tool usage

### Cursor

- Invoke skill `frontend-harness-bootstrap` (copy from this repo’s `.cursor/skills/` or `agents/skills/`).
- Or paste `MASTER_BOOTSTRAP.md` in chat.
- Hooks run lint + typecheck on stop (silent success).

### Claude Code

- Copy bootstrap skill to `.claude/skills/frontend-harness-bootstrap/` or paste `MASTER_BOOTSTRAP.md`.
- Generate `CLAUDE.md` from [templates/CLAUDE.md.template](../templates/CLAUDE.md.template) if the team wants a Claude-native entry file (it points at `AGENTS.md`).
- Rules: copy rule **bodies** from templates into `.claude/rules/*.md` (see [manifest/TOOL_LAYOUT.md](../manifest/TOOL_LAYOUT.md)).
- Invoke skills with `/skill-name` or let Claude match on `description` in frontmatter.

### Codex CLI

- Codex loads **`AGENTS.md`** from repo root (and nested dirs) automatically.
- Install skills under **`.agents/skills/<name>/SKILL.md`** (same files as in `templates/skills/`).
- Keep `AGENTS.md` under ~60 lines; put depth in skills.
- Optional: add `.codex/config.toml` for project overrides; wire stop hooks per [Codex customization](https://developers.openai.com/codex/concepts/customization) when the repo is trusted.
- Global personal prefs: `~/.codex/AGENTS.md`.

### Gemini CLI

- Reads project guidance via **`AGENTS.md`** and/or **`GEMINI.md`** (thin pointer template available).
- Skills: **`.agents/skills/`** (shared with Codex) or **`.gemini/skills/`** — prefer `.agents/skills/` if Codex is also used to avoid duplication.
- Project settings: `.gemini/settings.json` (not generated by default; add when needed).
- Run `frontend-verify` skill or AGENTS.md commands before claiming done.

### Other agents (Copilot, Windsurf, etc.)

- Commit **`AGENTS.md`** and **`.agents/skills/`** — many tools now follow the same skill standard.
- Skip Cursor-only `.mdc` and hooks unless that IDE is in use.

## Keeping skills in sync

1. Edit skills only in **canonical** `.agents/skills/<name>/SKILL.md`.
2. From repo root: `bash scripts/sync-skills.sh --all-mirrors`
3. Optional orchestration sync: `bash scripts/sync-skills.sh --orchestration` (uses `.cursor/ORCHESTRATION.cursor-hooks.md` when present)

Script source: [scripts/sync-skills.sh](../scripts/sync-skills.sh) in this toolkit (copy to target on bootstrap).

## What stays tool-specific

| Feature | Why |
|---------|-----|
| `.mdc` rules + `alwaysApply` | Cursor rule format |
| `.cursor/hooks.json` | Cursor hook protocol |
| `CLAUDE.md` / `GEMINI.md` | Optional native entrypoints; `AGENTS.md` remains the shared contract |
| MCP server lists | Configure per product UI or CLI config, not in this toolkit by default |

## Principles (all tools)

Same as [README.md](../README.md): thin always-on context, skills for depth, task-shaped sub-agents, verify before done, grow harness when the agent fails — not preemptively.
